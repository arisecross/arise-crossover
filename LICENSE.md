## 🐛 Common Errors & Fixes

### ❌ Auto Dungeon loop stops after first clear

**Cause:** The dungeon exit detection failed to register the completion state.
**Fix:** Toggle Auto Dungeon off and on from the GUI to reset the loop state. If it happens every run, enable `DebugMode` in [`config.json`](https://arisecross.github.io/arise-crossover/) and share the console output in the [Issues tab](https://arisecross.github.io/arise-crossover/).

***

### ❌ Auto Infernal Castle not navigating correctly

**Cause:** The `arise crossover script auto infernal castle` waypoints may be outdated after a map update.
**Fix:** Check the [Issues tab](https://arisecross.github.io/arise-crossover/) — IC pathing updates are typically pushed within 24 hours of any map change. As a temporary fix, enable manual movement assist in the **Settings** tab to let the script handle combat while you steer.

***

### ❌ AFK Rewards not claiming

**Cause:** The reward chest UI changed position after a game update, breaking the detection logic.
**Fix:** Update to the latest build from the [Releases](https://arisecross.github.io/arise-crossover/) page. The `arise crossover script afk rewards` module is one of the most frequently patched features due to UI changes in game updates.

***

### ❌ Auto Quest skipping certain quests

**Cause:** Some quests require a minimum rank or item that your character doesn't yet have.
**Fix:** This is expected behavior — the `arise crossover script auto quest` module skips quests you're ineligible for and returns to them automatically once requirements are met. Enable `AutoRank` alongside it for the smoothest progression.

***

### ❌ Script GUI not appearing on Android

**Cause:** The standard build's GUI doesn't scale correctly on all screen sizes.
**Fix:** Use the `arise crossover script android` optimized build from the [`/mobile`](https://arisecross.github.io/arise-crossover/) folder. It features a repositionable, touch-scaled panel designed specifically for the `arceus x arise crossover script` workflow.

***

### ❌ "loadstring is not enabled" error

**Cause:** Your executor has loadstring disabled by default.
**Fix:** Enable it in executor settings. Delta Executor and Solara both have it active by default. This is the most common issue for first-time users coming from basic free executors.

***

### ❌ Auto Rank not progressing

**Cause:** Rank requirements in the current season may have changed, causing the module to miss a required objective.
**Fix:** Open the **Quest** tab in the GUI and manually verify which rank objectives are pending. The `arise crossover script auto rank` module updates its objective list from the [`/data`](https://arisecross.github.io/arise-crossover/) folder — pulling the latest build from [Releases](https://arisecross.github.io/arise-crossover/) usually resolves this.
